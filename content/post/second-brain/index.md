## 🌿 人類的大腦是用來「產生想法」
大腦不僅僅是「儲存資訊」。在資訊爆炸的時代，若僅依賴我們的記性，你的創意將被瑣碎的待辦事項與未經整理的資料淹沒。Tiago Forte 提出的「第二大腦」系統，旨在建立一個外部的數位系統，幫你捕捉、組織並檢索所有知識。

## 🍇 框架與系統

### 1. CODE 框架：知識處理的流水線
一套完整的知識管理流程應包含以下四個階段：

- **捕捉 (Capture)**：只保留對你有共鳴、有價值的資訊。拒絕「囤積」，只收錄能觸動內心的內容。

- **組織 (Organize)**：根據「用途」而非「來源」進行分類。這就是 PARA 系統的用武之地。

- **萃取 (Distill)**：透過「漸進式總結 (Progressive Summarization)」，在不同時間點對筆記進行去蕪存菁，確保未來檢索時能秒懂重點，假設看了一本好書，第一次的筆記跟第五次看的心得一定不一樣，應該會越來越完整。簡單來說漸進式總結就是隨時記、隨時寫、隨時改，不用想說一次到位。

- **表達 (Express)**：知識的終點是產出。將儲存的「中間產物 (Intermediate Packets)」組合成文章、影片或專案成果。

### 2. PARA 系統：以行動為核心的分類法
不要按學科分類（如：經濟學、心理學），要按**行動狀態**分類：

- **專案 (Projects)**：具有明確目標與截止日期的短期任務（如：完成 Hugo 網站架設）。
- **領域 (Areas)**：需要長期維護且負責任的範疇（如：健康、財務、Maritime Engineering）。
- **資源 (Resources)**：感興趣但目前無特定行動的持續性主題（如：ComfyUI 研究、AI 演算法）。
- **封存 (Archives)**：已完成或不再活躍的上述三類項目。

## 🍷 小結


1. **拒絕過度組織 (Over-engineering)**：不要在還沒產出前就建立複雜的資料夾結構。Obsidian 的雙向連結 [[Internal Links]] 應該是為了連結想法，而非單純分類。
2. **產出導向**：如果你收集了 100 篇關於 AI 的文章卻沒寫出一行程式碼或一篇分析，那不叫第二大腦，那叫「數位垃圾場」。
3. **工具只是輔助**：無論是 Obsidian 還是 Hugo，都只是實現 CODE 流程的工具。系統邏輯（System Logic）永遠優先於工具功能。
